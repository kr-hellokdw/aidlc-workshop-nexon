# 유저 스토리 (User Stories)

분류: Persona 기반 | 세분화: 주요 기능별 | 수용 기준: 체크리스트 | 우선순위: 사용자 여정 순서

---

## 고객 스토리 (Customer Stories)

### US-C01: 태블릿 자동 로그인
**As a** 고객,  
**I want** 테이블에 앉으면 별도 조작 없이 주문 화면이 바로 표시되기를,  
**So that** 즉시 메뉴를 탐색하고 주문할 수 있다.

**수용 기준:**
- [ ] 태블릿 브라우저 접속 시 저장된 인증 정보로 자동 로그인된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 기본으로 표시된다
- [ ] 자동 로그인 실패 시 (미설정/만료) 설정 화면으로 이동한다
- [ ] 로그인 상태에서 매장 식별자와 테이블 번호가 올바르게 연결된다

**Persona**: 고객 (김민수)

---

### US-C02: 메뉴 조회 및 탐색
**As a** 고객,  
**I want** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를,  
**So that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다.

**수용 기준:**
- [ ] 메뉴가 카테고리별로 분류되어 표시된다
- [ ] 카테고리 탭 클릭 시 해당 카테고리 메뉴로 빠르게 이동한다
- [ ] 각 메뉴에 메뉴명, 가격, 설명, 이미지가 표시된다
- [ ] 카드 형태 레이아웃으로 시각적으로 구분된다
- [ ] 터치 영역이 최소 44x44px 이상이다

**Persona**: 고객 (김민수)

---

### US-C03: 장바구니 관리
**As a** 고객,  
**I want** 주문할 메뉴를 장바구니에 담고 수량을 조절하기를,  
**So that** 주문 전에 선택한 메뉴를 확인하고 수정할 수 있다.

**수용 기준:**
- [ ] 메뉴 선택 시 장바구니에 추가된다
- [ ] 장바구니에서 수량 증가/감소가 가능하다
- [ ] 장바구니에서 특정 메뉴를 삭제할 수 있다
- [ ] 총 금액이 실시간으로 계산되어 표시된다
- [ ] 장바구니 전체 비우기가 가능하다
- [ ] 페이지 새로고침 후에도 장바구니 내용이 유지된다
- [ ] 장바구니가 비어있을 때 적절한 안내 메시지가 표시된다

**Persona**: 고객 (김민수)

---

### US-C04: 주문 생성
**As a** 고객,  
**I want** 장바구니의 메뉴를 최종 확인하고 주문을 확정하기를,  
**So that** 선택한 메뉴가 주방에 전달된다.

**수용 기준:**
- [ ] 주문 확정 전 주문 내역(메뉴, 수량, 금액)을 최종 확인할 수 있다
- [ ] 주문 확정 버튼 클릭 시 서버에 주문이 전송된다
- [ ] 주문 성공 시 주문 번호가 5초간 표시된다
- [ ] 주문 성공 후 장바구니가 자동으로 비워진다
- [ ] 주문 성공 후 메뉴 화면으로 자동 리다이렉트된다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다
- [ ] 주문에 매장 ID, 테이블 ID, 세션 ID가 포함된다

**Persona**: 고객 (김민수)

---

### US-C05: 주문 내역 조회
**As a** 고객,  
**I want** 현재 테이블에서 주문한 내역을 확인하기를,  
**So that** 주문 상태와 총 주문 금액을 파악할 수 있다.

**수용 기준:**
- [ ] 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)
- [ ] 주문이 시간 순으로 정렬된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴/수량, 금액이 표시된다
- [ ] 주문 상태(대기중/준비중/완료)가 표시된다
- [ ] 이용 완료 처리된 주문은 표시되지 않는다

**Persona**: 고객 (김민수)

## 관리자 스토리 (Admin Stories)

### US-A01: 매장 관리자 로그인
**As a** 매장 관리자,  
**I want** 매장 식별자와 계정 정보로 관리 시스템에 로그인하기를,  
**So that** 매장 운영 기능에 접근할 수 있다.

**수용 기준:**
- [ ] 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있다
- [ ] 로그인 성공 시 주문 모니터링 대시보드로 이동한다
- [ ] 로그인 세션이 16시간 유지된다
- [ ] 브라우저 새로고침 시 세션이 유지된다
- [ ] 16시간 후 자동 로그아웃된다
- [ ] 잘못된 인증 정보 입력 시 에러 메시지가 표시된다
- [ ] 로그인 시도 횟수가 제한된다

**Persona**: 매장 관리자 (박영희)

---

### US-A02: 실시간 주문 모니터링
**As a** 매장 관리자,  
**I want** 들어오는 주문을 실시간으로 확인하고 상태를 관리하기를,  
**So that** 주문을 놓치지 않고 빠르게 처리할 수 있다.

**수용 기준:**
- [ ] 테이블별 카드 형태의 그리드 대시보드가 표시된다
- [ ] 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
- [ ] 신규 주문 발생 시 2초 이내에 화면에 반영된다 (SSE)
- [ ] 신규 주문이 시각적으로 강조된다 (색상/애니메이션)
- [ ] 주문 카드 클릭 시 전체 메뉴 목록 상세 보기가 가능하다
- [ ] 주문 상태를 대기중/준비중/완료로 변경할 수 있다
- [ ] 테이블별 필터링이 가능하다

**Persona**: 매장 관리자 (박영희)

---

### US-A03: 테이블 초기 설정
**As a** 매장 관리자,  
**I want** 테이블 태블릿의 초기 설정을 수행하기를,  
**So that** 고객이 해당 테이블에서 바로 주문할 수 있다.

**수용 기준:**
- [ ] 테이블 번호를 설정할 수 있다
- [ ] 테이블 비밀번호(기기 잠금용)를 설정할 수 있다
- [ ] 설정 완료 시 16시간 세션이 생성된다
- [ ] 설정 정보가 저장되고 자동 로그인이 활성화된다
- [ ] 설정 성공/실패 피드백이 표시된다

**Persona**: 매장 관리자 (박영희)

---

### US-A04: 주문 삭제
**As a** 매장 관리자,  
**I want** 잘못된 주문을 삭제하기를,  
**So that** 정확한 주문 현황을 유지할 수 있다.

**수용 기준:**
- [ ] 특정 주문에 삭제 버튼이 표시된다
- [ ] 삭제 클릭 시 확인 팝업이 표시된다
- [ ] 확인 후 주문이 즉시 삭제된다
- [ ] 삭제 후 해당 테이블의 총 주문액이 재계산된다
- [ ] 삭제 성공/실패 피드백이 표시된다

**Persona**: 매장 관리자 (박영희)

---

### US-A05: 테이블 이용 완료 처리
**As a** 매장 관리자,  
**I want** 고객이 떠난 테이블의 세션을 종료하기를,  
**So that** 새 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다.

**수용 기준:**
- [ ] 테이블 이용 완료 버튼이 표시된다
- [ ] 클릭 시 확인 팝업이 표시된다
- [ ] 확인 후 해당 세션의 주문 내역이 과거 이력으로 이동한다
- [ ] 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다
- [ ] 새 고객의 첫 주문 시 새로운 세션이 시작된다
- [ ] 성공/실패 피드백이 표시된다

**Persona**: 매장 관리자 (박영희)

---

### US-A06: 과거 주문 내역 조회
**As a** 매장 관리자,  
**I want** 테이블별 과거 주문 내역을 조회하기를,  
**So that** 이전 매출과 주문 이력을 확인할 수 있다.

**수용 기준:**
- [ ] 과거 내역 버튼 클릭 시 과거 주문 목록이 표시된다
- [ ] 테이블별로 과거 주문이 시간 역순으로 정렬된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
- [ ] 날짜 필터링이 가능하다
- [ ] 닫기 버튼으로 대시보드로 복귀할 수 있다

**Persona**: 매장 관리자 (박영희)

---

### US-A07: 메뉴 관리
**As a** 매장 관리자,  
**I want** 메뉴를 등록/수정/삭제하고 순서를 조정하기를,  
**So that** 고객에게 최신 메뉴 정보를 제공할 수 있다.

**수용 기준:**
- [ ] 카테고리별로 메뉴 목록을 조회할 수 있다
- [ ] 새 메뉴를 등록할 수 있다 (메뉴명, 가격, 설명, 카테고리, 이미지)
- [ ] 기존 메뉴를 수정할 수 있다
- [ ] 메뉴를 삭제할 수 있다
- [ ] 메뉴 노출 순서를 조정할 수 있다
- [ ] 필수 필드 미입력 시 검증 에러가 표시된다
- [ ] 가격이 유효 범위를 벗어나면 검증 에러가 표시된다
- [ ] 이미지 파일을 업로드할 수 있다

**Persona**: 매장 관리자 (박영희)
