# 사용자 페르소나 (User Personas)

---

## Persona 1: 고객 (Customer)

### 프로필
| 항목 | 내용 |
|------|------|
| **이름** | 김고객 (대표 고객) |
| **역할** | 매장 방문 고객 |
| **연령대** | 20~50대 (폭넓은 연령층) |
| **기술 수준** | 초~중급 (스마트폰/태블릿 기본 사용 가능) |

### 목표 (Goals)
- 빠르고 간편하게 음식을 주문하고 싶다
- 메뉴를 충분히 살펴보고 선택하고 싶다
- 주문 상태를 확인하고 싶다
- 직원을 부르지 않고도 추가 주문을 하고 싶다

### 동기 (Motivations)
- 대기 시간 최소화
- 편리한 주문 경험
- 주문 실수 방지 (직접 확인 후 주문)

### 불편 사항 (Pain Points)
- 직원을 기다리는 시간이 길다
- 메뉴판이 잘 안 보이거나 정보가 부족하다
- 주문이 제대로 들어갔는지 불안하다

### 사용 환경
- 매장 테이블에 설치된 태블릿
- 터치 기반 인터랙션
- 식사 중 사용 (손이 깨끗하지 않을 수 있음)

## Persona 2: 매장 관리자 (Store Admin)

### 프로필
| 항목 | 내용 |
|------|------|
| **이름** | 박사장 (대표 관리자) |
| **역할** | 매장 사장/매니저 |
| **연령대** | 30~60대 |
| **기술 수준** | 초~중급 (기본 PC/모바일 사용 가능, IT 전문가 아님) |

### 목표 (Goals)
- 들어오는 주문을 실시간으로 파악하고 싶다
- 테이블별 주문 현황을 한눈에 보고 싶다
- 메뉴를 쉽게 관리(추가/수정/삭제)하고 싶다
- 매장 운영을 효율적으로 하고 싶다

### 동기 (Motivations)
- 인건비 절감 (주문 접수 인력 감소)
- 주문 누락/실수 방지
- 빠른 주문 처리로 고객 만족도 향상
- 매장 운영 데이터 확보

### 불편 사항 (Pain Points)
- 바쁜 시간대에 주문을 놓치는 경우가 있다
- 수기 주문 시 실수가 발생한다
- 테이블별 주문 현황 파악이 어렵다
- 메뉴 변경 시 메뉴판 교체 비용/시간이 든다

### 사용 환경
- PC 또는 태블릿 (카운터/사무실)
- 브라우저 기반 관리 화면
- 영업 시간 내 지속 사용
